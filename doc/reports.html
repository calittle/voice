Project Performance Report

VOICE - Vote Online Interactive Civic Enablement

Project Manager: Charles Little

Date Prepared: 24 June 2017

Sponsor: Dr. Hayden Wimmer

Accomplishments for This Reporting Period:
  • Started and completed functional requirements definition after review of voting registration requirements for the state of Georgia, and survey of similar requirements for other states.
  • Started and completed technical requirements definition based on initial project proposal and desired end-state of the system.
  • Generated work breakdown structure (WBS) based on technical and functional requirements and the needs of the project.
  • Started and completed design of Data Model as iterative process. Initial data model was refined and developed to final model.
  • Started and completed hardware procurement decision process. Reviewed cost, time, and complexities of using IaaS or SaaS offerings from Amazon Web Services and 1and1 hosting, versus construction/configuration of hardware/software at home office. Decision was taken for to perform construction/configuration option for maximum flexibility and availability of existing hardware components to be repurposed at no cost.
  • Completed solution design based on technical and functional requirements.
  • Installed LAMP stack using Ubuntu 16.04, Apache, MySQL, and PHP.
  • Started development work:
    • Created and tested DDL scripts to create schema entities and relations.
    • Created and tested DML scripts to generate stored procedures according to design.
    • Created and tested DML scripts to insert static configuration data into schema.
    • Created and tested DML scripts for unit testing stored procedures.
    • Started construction of HTML/PHP pages
Accomplishments Planned but not Completed This Reporting Period:
  • None.
Root Cause of Variances
  • Not applicable; plan is currently on schedule.
Planned Corrective or Preventive Action
  • Review remaining tasks and durations in project plan and compare to delivery date. Adjust working hours per day to crash schedule, or review scope for task removal if plan may miss delivery date.
Planned Accomplishments for Next Reporting Period
  • Complete development work by 30 June 2017.
  • Start and complete Unit Testing by 7 July 2017.
  • Complete final project documentation and delivery by 9 July 2017
New Risks Identified
  • Developer has concerns that PHP development process may be slowed due to learning secure development practices for that platform. Will work to mitigate by crashing project plan by extending work day hours.
  • Developer has concerns that Javascript development process may be slowed due to selection of Bootstrap framework, which utilizes JQuery. Will work to mitigate by trying to use provided functions of JQuery rather than developing from scratch.
Issues
  • Hardware platform on initial build exhibited stability issues with X graphical user interface and NVidia Geforce drivers for Ubuntu. Resolved by reinstalling Ubuntu with headless server configuration and no X window interface.
  • Final solution design centralizes business logic in stored procedures at data layer (initial design was to use logic in middleware layer, e.g. Java application server classes). Design change results in work breakdown structure shift to require a MySQL developer instead of Java developer.
  • Unfamiliarity with MySQL development conventions slowed initial development. Resolved by crashing project plan by adding more hours to work day to compensate.
Comments
  • Project is progressing well despite initial build issues and overall complexity. Expect project will conclude with scope successfully met
